# 韓国語翻訳ガイド — 번역체を避ける

atfedi.de の韓国語（`ko`）を書く・直すときの指針。

> 三言語に共通する「構造の文化」（どこに重心を置くか）は
> [`writing.md`](./writing.md) に。ここは韓国語に固有の話。

## なぜこれがあるか

atfedi.de の韓国語は、最初に日本語をもとに訳した。日本語と韓国語は語順も
文法もよく似ているため、直訳しても「通じてしまう」。だからこそ
**번역체（翻訳体）** — 文法的には正しいが翻訳くさい、不自然な韓国語 —
が紛れ込みやすい。

> **注記:** このガイドは韓国語ネイティブの校正の代わりにはならない。
> 公開前に一度、ネイティブの目を通すのが望ましい。

## 落とし穴

### 1. 「の」→「의」の連鎖・濫用

日本語の「の」をすべて「의」にすると、「의」が積み重なって不自然になる。
韓国語は「의」を省くか、動詞・連体形で言い換える。

- ✗ 소셜 플랫폼들의 네트워크 → ○ 소셜 플랫폼이 모인 네트워크
- ✗ 어느 한 회사의 것（の＋もの）→ ○ 어느 한 회사도 소유하지 않는다

### 2. 複数の「들」濫用（〜たち・複数）

日本語の「〜たち」や英語の複数を機械的に「들」にしない。文脈で複数と
わかるなら「들」は付けない。「들」＋「의」の重なりは特に翻訳くさい。

- ✗ 독립된 플랫폼들의 / 도구들을 / 계정들
- ○ 독립된 플랫폼이 / 도구를 / 계정

ただし禁止ではない。主語の「複数であること」自体が主題のときは
「들」が自然（서버들은 서로 대화하는 방식은 맞춰져 있지만 —
サーバ**同士**の話をしている）。機械的に付けない・機械的に剥がさない。

### 3. 受動の濫用（〜される → 〜되어지다 / 〜되다）

韓国語は能動が基本。二重受動「〜되어지다」は誤り。

- ✗ 사용되어지다, 만들어지다（の濫用）→ ○ 쓰다, 만들다（能動で）
- ✗ 누군가에 의해 만들어진 → ○ 누군가가 만든

### 4. 「〜について / 〜に対して」→「〜에 대해 / 대한」濫用

- ✗ 페디버스에 대한 도구 → ○ 페디버스 도구 / 페디버스를 위한 도구

### 5. 「あなた」→「당신」

韓国語の UI で「당신」はよそよそしく響く。二人称は基本的に省く。

- ✗ 당신의 계정 → ○ 내 계정 ／ 계정（省略）

### 6. 「〜ている」→「〜고 있다」濫用

状態・習慣は現在形で足りることが多い。

- ✗ ActivityPub를 지원하고 있습니다 → ○ ActivityPub를 지원합니다

### 7. 漢字語より、自然な固有語を

日本語の漢字語をそのまま音で移さず、自然な固有語があればそちらを選ぶ。

- 道しるべ → 「안내서」より「길잡이」

### 8. 文末の語調（register）を統一する ← いちばん大事

語調の混在が、いちばん「翻訳くさい」。1つの種類のテキストの中で
「〜습니다」「〜다」「体言止め」が混ざらないようにする。

2026-07 の vlt 記事のネイティブ磨きは、直しがこの項の三箇所だけだった。
해요체の本文に紛れ込んだ体言止めを、述語で言い切る:

- 지금 버전은 1.0.0-rc.32. → 1.0.0-rc.32**예요**.
- 둘 다 95MB. → 95MB**였어요**.
- (친절과 오지랖의 딱 중간) → (딱 중간**이에요**)

括弧の中の名詞句も例外にしない。§9（言い差し）と同族で、日本語の
体言止めは 해요체 の流れの中では文が未完成に聞こえる。

### 9. 日本語の「言い差し」— 省いた後ろを、そのまま運ばない

日本語は高文脈で（→ [`writing.md`](./writing.md) の「学校嫌い」）、決まり
文句では後ろ — 多くは述語 — を省き、聞き手が連想で補う。省略したかたちが、
そのまま挨拶・定型句として定着している。

- よろしく ＜ よろしく（お願いします）
- すみません ＜ すみません（が、ちょっと…）
- また ＜ また（会いましょう）
- こんにちは ＜ 今日は（ご機嫌いかがですか）

韓国語は、この「言い差し」の習慣が弱い。省いたまま訳すと、文が途中で折れて
聞こえる。**省かれた語、とくに文末の述語を補って訳す。**

- ✗ 다음 이야기는, 또. — 「つづきは、また。」の直訳。「또」が宙づりになる
- ○ 다음 이야기에서 또 만나요. — 述語「만나요」で締める

韓国語の挨拶は、もともと述語まで言い切る形が多い（안녕하세요 / 또 만나요 /
또 봬요）。日本語の「また」「よろしく」のような裸の語を、そのまま「또」「잘」
で置かない。

「まだです」も同族。「README도 아직이에요」は宙づりに聞こえる —
「README도 아직 없어요」と、何が「まだ」なのかを述語で言い切る。

### 10. 動詞は、目的語との韓国語慣用で選び直す（カルク回避）

日本語の動詞・比喩をそのまま movす と、語は通じても組み合わせが翻訳くさい。
目的語に対して韓国語が慣用的に選ぶ動詞に置き換える。2026-07 の
activityplug 記事をネイティブ調に磨いてもらった差分から:

- 中を歩いてみたら → ✗ 안을 걸어 보니 → ○ 막상 안을 **들여다보니**
- base64 を開けると → ✗ base64를 열면 → ○ base64를 **풀어보면**（decode=풀다）
- 持ち方が二つ → ✗ 쥐는 방법이 두 가지 → ○ **쓰는** 방법은 두 가지
- 実行時 → ✗ 실행 시 → ○ 실행 **시점**
- 下は REST（実装の下側）→ ✗ 아래는 REST → ○ **내부는** REST
- （サーバに）触るもの → ✗ 만지는 것 → ○ **다루는** 무언가
- 設計の真ん中を通る → ✗ 한가운데를 지나다 → ○ 한가운데를 **꿰뚫다**
- 別な設計をしている → ✗ 설계를 하고 있어요 → ○ 설계를 **갖고** 있어요

### 11. 副詞の潤滑油を足す

日本語より一段多く、막상・바로・직접・곧바로・미리・함께・그대로 のような
副詞で文の流れに手触りを付ける。無いと文法的に正しくても平板で、
それが「翻訳くさい」の正体の一つ。

- 알려 주면 통해요 → **직접** 알려주면 **곧바로** 통해요
- 타입이 막아 줘요 → 타입 시스템이 **미리** 막아줘요

### 12. 引数の明示化を、もう一歩

§9（言い差し）の続き。名詞・引用も省略形のまま運ばず、完全な形にする。

- 타입이 막아 줘요 → **타입 시스템이** 막아줘요（何のタイプか言い切る）
- 세 값 → 세 **가지** 값（数詞には数え言葉）
- 引用は完全な発話に: ✗ "Mastodon으로 대해 주세요"를 받는다
  → ○ **"저를 Mastodon으로 대해주세요"라고 말하는 걸** 받는다
- 話し言葉の引用は直訳せず自然な口語に: 「このサーバは誰？」
  → ✗ "이 서버는 누구야?" → ○ "이 서버, **정체가 뭐야?**"

### 13. 보조용언は、blog の 해요체 では付けて書く

표준어 규정は 띄어쓰기 原則・붙여쓰기 許容。カジュアルな 해요체 の文章では
붙여쓰기（걸어본 / 꽂아봤어요 / 열어보면）の方が柔らかく読める。
どちらでもよいが、**一つの記事の中では統一**する。

### 14. こまごま

- 「AとBと、C」の「〜と、」を運ばない: ✗ 로그인과, 정직한 에러
  → ○ 로그인, **그리고** 정직한 에러
- 丁寧な 해요체 の中の「うちの」は 우리 より **저희**
- 術語の初出は易しい우리말で導入し（capability 목록표 → **기능** 목록표）、
  以後の精密な議論では原語術語に戻ってよい
- 記事タイトルは体言止めが自然: 「〜와 함께 걸어 봤어요」より
  「〜와 함께 걸어본 **이야기**」

### 15. 術語の初出に、短い우리말の括弧注を

§12・§14 の続き。2026-07 leaving-fedify のネイティブ磨き差分から。
韓国語の技術文は、術語の初出に一般読者向けの短い풀이を括弧で添えるのが親切:

- 멱등성(같은 요청이 여러 번 들어와도 결과가 한 번만 반영되게 하는 장치)
- OOM(메모리가 부족해 죽는 것) / raw body(가공하지 않은 원본 데이터) /
  JWK(공개키를 담는 표준 형식) / 골든 픽스처(정답 데이터)
- 풀이が一文ぶんあるなら줄표も可（§「줄표は稀」の例外）:
  결정적이고 — 같은 입력이면 언제나 같은 출력이 나온다는 뜻이에요 — …도 결정적이에요

同じ差分から、あわせて:

- 外来語も自然な우리말へ（§7 の外来語版）: 프리미티브 → **기본 부품**。
  二度目からは「**그것**만이라면」のような指示語で受けてよい
- 名詞化をほどいて動詞で言う: ✗ 서명의 구축과 검증을 따라간다
  → ○ 서명을 **만들고 검증하는 과정**을 따라간다
- 強調の助詞・副詞が太字の代わりになる: HTTP 서버**조차** 없는 /
  **다름 아닌** fedify 자신이었어요（後者は文の太字を外してよい）
- 数詞には数え言葉（§12 再掲）: 둘 다 → **두 가지** 서명 방식을 다

### 16. 文末語尾に変化をつける（〜어요 の単調を破る）

2026-07 server-rail-map のネイティブ磨き差分から。シロの下書きは文末が
ほぼ全部「〜어요/〜해요」で、文法的には正しいが平板だった。ネイティブは
記事全体で語尾を三種類ほど使い分けていた:

- **〜더라고요** — 自分で見て気づいたこと・意外だった発見の回想:
  곧바로 바깥으로 나가**더라고요** / 얼마든지 같이 갈 수 있**더라고요** /
  거짓말이 되지 않**더라고요**
- **〜죠 / 〜는 거죠** — 読者と共有できる前提・結論の念押し:
  보통 상태 페이지나 대시보드**죠** / 진짜여야 한다는 **거죠** /
  골라야 하는 값이었던 **거죠** / 한눈에 알아차릴 수 있는 **거죠**
- **〜어요** — それ以外の地の文

使い分けの軸は「その文の情報が誰のものか」。自分の発見なら 더라고요、
読者も頷ける話なら 죠、ただの叙述なら 어요。§11 の副詞と同じで、
無くても通じるが、無いと「翻訳くさい」の正体になる。

同じ差分から、こまごま:

- 感情・評価も一語添えて明示する（§12 の感情版）: 로켓이 뒤로 날아가는
  **우스운 일이 생겼어요**（日本語版は「後ろ向きに飛びました」で終わり）
- 「報告が来た」のような硬い受けは **제보가 들어왔죠** のような
  こなれた語彙に

## atfedi.de の韓国語スタイル

| テキストの種類 | 語調 | 例 |
|---|---|---|
| UIラベル（ナビ・ボタン・カテゴリ名） | 体言止め | `전체` `웹사이트` `브리지` |
| 見出し（headline・heading） | 体言止め | `페디버스 둘러보기` |
| 見出し・タグライン | 体言止め | `페디버스 길잡이` |
| 説明文（intro） | 해요체 | `〜예요` `〜할 수 있어요` |
| ツールの要約（summary） | 해요체、述語で締める | `〜클라이언트예요.` `〜할 수 있어요.` |

固有名詞:

- プラットフォーム名は定着した音写を使う — 마스토돈, 미스키, 픽셀페드
- プロトコル名・言語名・製品名はラテン文字のまま — ActivityPub, TypeScript, Python, MFM

句読点:

- 「」（홑낫표）は韓国語では使わない。引用は " "
- 中黒「·」は併記に使ってよい（앱·클라이언트）
