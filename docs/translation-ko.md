# 韓国語翻訳ガイド — 번역체を避ける

atfedi.de の韓国語（`ko`）を書く・直すときの指針。

> 三言語に共通する「構造の文化」（どこに重心を置くか）は
> [`writing.md`](./writing.md) に。ここは韓国語に固有の話。

## なぜこれがあるか

atfedi.de の韓国語は、最初に日本語をもとに訳した。日本語と韓国語は語順も
文法もよく似ているため、直訳しても「通じてしまう」。だからこそ
**번역체（翻訳体）** — 文法的には正しいが翻訳くさい、不自然な韓国語 —
が紛れ込みやすい。

> **注記:** このガイドは韓国語ネイティブの校正の代わりにはならない。
> 公開前に一度、ネイティブの目を通すのが望ましい。

## 落とし穴

### 1. 「の」→「의」の連鎖・濫用

日本語の「の」をすべて「의」にすると、「의」が積み重なって不自然になる。
韓国語は「의」を省くか、動詞・連体形で言い換える。

- ✗ 소셜 플랫폼들의 네트워크 → ○ 소셜 플랫폼이 모인 네트워크
- ✗ 어느 한 회사의 것（の＋もの）→ ○ 어느 한 회사도 소유하지 않는다

### 2. 複数の「들」濫用（〜たち・複数）

日本語の「〜たち」や英語の複数を機械的に「들」にしない。文脈で複数と
わかるなら「들」は付けない。「들」＋「의」の重なりは特に翻訳くさい。

- ✗ 독립된 플랫폼들의 / 도구들을 / 계정들
- ○ 독립된 플랫폼이 / 도구를 / 계정

### 3. 受動の濫用（〜される → 〜되어지다 / 〜되다）

韓国語は能動が基本。二重受動「〜되어지다」は誤り。

- ✗ 사용되어지다, 만들어지다（の濫用）→ ○ 쓰다, 만들다（能動で）
- ✗ 누군가에 의해 만들어진 → ○ 누군가가 만든

### 4. 「〜について / 〜に対して」→「〜에 대해 / 대한」濫用

- ✗ 페디버스에 대한 도구 → ○ 페디버스 도구 / 페디버스를 위한 도구

### 5. 「あなた」→「당신」

韓国語の UI で「당신」はよそよそしく響く。二人称は基本的に省く。

- ✗ 당신의 계정 → ○ 내 계정 ／ 계정（省略）

### 6. 「〜ている」→「〜고 있다」濫用

状態・習慣は現在形で足りることが多い。

- ✗ ActivityPub를 지원하고 있습니다 → ○ ActivityPub를 지원합니다

### 7. 漢字語より、自然な固有語を

日本語の漢字語をそのまま音で移さず、自然な固有語があればそちらを選ぶ。

- 道しるべ → 「안내서」より「길잡이」

### 8. 文末の語調（register）を統一する ← いちばん大事

語調の混在が、いちばん「翻訳くさい」。1つの種類のテキストの中で
「〜습니다」「〜다」「体言止め」が混ざらないようにする。

## atfedi.de の韓国語スタイル

| テキストの種類 | 語調 | 例 |
|---|---|---|
| UIラベル（ナビ・ボタン・カテゴリ名） | 体言止め | `전체` `웹사이트` `브리지` |
| 見出し（headline・heading） | 体言止め | `페디버스 둘러보기` |
| 見出し・タグライン | 体言止め | `페디버스 길잡이` |
| 説明文（intro） | 해요체 | `〜예요` `〜할 수 있어요` |
| ツールの要約（summary） | 해요체、述語で締める | `〜클라이언트예요.` `〜할 수 있어요.` |

固有名詞:

- プラットフォーム名は定着した音写を使う — 마스토돈, 미스키, 픽셀페드
- プロトコル名・言語名・製品名はラテン文字のまま — ActivityPub, TypeScript, Python, MFM

句読点:

- 「」（홑낫표）は韓国語では使わない。引用は " "
- 中黒「·」は併記に使ってよい（앱·클라이언트）
